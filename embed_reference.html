Volcano Dashboard Embedding Guide

This guide shows how to properly embed the Volcano Dashboard in your website.

Basic Embedding

To embed the entire dashboard, use this code:

<iframe 
    src="YOUR_APP_URL" 
    width="100%" 
    height="800px" 
    style="border:none;margin:0;padding:0;" 
    allow="fullscreen"
    title="Volcano Monitoring Dashboard">
</iframe>
Note: Replace YOUR_APP_URL with your actual Replit app URL.

Embedding Specific Pages

To embed specific pages from the dashboard, add the page name to the URL:

Scientific 3D Eruption

<iframe 
    src="YOUR_APP_URL/scientific_3d_eruption" 
    width="100%" 
    height="800px" 
    style="border:none;margin:0;padding:0;" 
    allow="fullscreen"
    title="Scientific 3D Volcano Eruption">
</iframe>

Optimization for Embedded Views

When embedding, consider the following optimization techniques:

  • Use the Low quality setting in embedded views to improve performance
  • Set appropriate height - 700px to 800px is recommended for the 3D visualizations
  • If you're embedding multiple visualizations, use a minimum height of 500px

Troubleshooting

If you encounter any issues with embedding:

  1. Make sure CORS is enabled on the server (already configured in this app)
  2. Ignore 404 errors for /_stcore/health and /_stcore/host-config - these are normal when embedding
  3. If visualizations don't load, try adding the following parameter: ?embed=true to the URL
Performance Tip: The Scientific 3D Eruption page is the most resource-intensive. For best performance when embedding, consider using the Low quality setting, which can be set in the sidebar after the page loads.
